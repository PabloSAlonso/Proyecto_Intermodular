# Development Rules

Generales:

- usar arquitectura limpia
- separar lógica de negocio
- evitar código duplicado
- escribir código legible
- comentar funciones importantes

PHP:

- usar MVC si es posible
- sanitizar inputs
- usar PDO para base de datos
- php puro para api

Java Android:

- usar arquitectura MVVM
- separar UI de lógica

C#:

- usar arquitectura por capas
- usar HttpClient para API

API:

- respuestas JSON
- manejo correcto de errores
