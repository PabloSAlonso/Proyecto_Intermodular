# README – Proyecto Intermodular  

## 1. Introducción  

Este documento actúa como diario de desarrollo del proyecto intermodular.  

El objetivo principal es la creación de tres aplicaciones a lo largo del curso:

- Aplicación web  
- Aplicación móvil  
- Aplicación de escritorio  

En este diario se registra, semana a semana, el trabajo realizado, las decisiones tomadas, los problemas encontrados y las soluciones aplicadas durante el desarrollo.

La finalidad es documentar el progreso del proyecto, facilitar su seguimiento y disponer de una referencia futura que permita entender cómo y por qué se implementaron determinadas funcionalidades.

## 2. Desarrollo por semanas  

### Enero  

#### 1ª semana
- Diseño web de la página de inicio de sesión.
- Diseño web de la página de registro.
- Creación de la base de datos.
- Creación de las tablas principales:
  - `usuarios`
  - `publicaciones`
  - `historias`

#### 2ª semana
- Diseño móvil de las páginas de inicio de sesión y registro.
- Implementación de Bootstrap para la maquetación de la aplicación web.

#### 3ª semana
- Creación de clases en Java para la aplicación móvil:
  - Gestión de objetos `Usuario`
  - Gestión de objetos `Publicacion`
- Implementación de la estructura MVC en la aplicación web.
- Creación de las primeras clases modelo.
- Diseño de un logotipo común para las tres aplicaciones.

#### 4ª semana
- Inicio del diseño de las celdas de publicaciones en la aplicación móvil.
- Implementación de `RecyclerView` y su adaptador.
- Finalización del `RecyclerView` (pendiente integrar funcionalidades de la API).
- Estructuración del feed principal en la aplicación web.
- Creación de la clase de conexión a la base de datos.
- Creación de nuevas tablas:
  - `likes_publicaciones`
  - `likes_usuarios`
